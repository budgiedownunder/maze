## Source format and PNG generation

Diagrams are authored in [PlantUML](https://plantuml.com/) using the [C4-PlantUML](https://github.com/plantuml-stdlib/C4-PlantUML) library. The `*.puml` source files live alongside their rendered PNGs in this directory.

PNGs are auto-regenerated by the [`generate-png-from-puml.yml`](../../.github/workflows/generate-png-from-puml.yml) GitHub Actions workflow whenever a pull request modifies any `**/*.puml` file — the workflow commits the updated PNGs back to the PR branch automatically. It can also be triggered manually from the Actions tab. **You don't need to commit PNGs locally when editing a `.puml` source** — they will be regenerated on the PR.

The workflow runs PlantUML `v1.2024.6` with Graphviz on `ubuntu-latest`.

## Local preview

To preview a diagram before pushing:

- **VS Code** — you can choose to install the [PlantUML extension](https://marketplace.visualstudio.com/items?itemName=jebbs.plantuml) for in-editor preview (`Alt+D`) or some other extension as preferred.
- **Command line** — if you have a `plantuml` wrapper on `PATH` (e.g. via [Chocolatey](https://community.chocolatey.org/packages/plantuml), [Scoop](https://scoop.sh/), or `apt install plantuml`):

  ```bash
  plantuml docs/diagrams/architecture.puml
  plantuml docs/diagrams/components.puml
  ```

  Otherwise, run the jar directly (this is what CI does):

  ```bash
  java -jar plantuml.jar -tpng docs/diagrams/architecture.puml
  java -jar plantuml.jar -tpng docs/diagrams/components.puml
  ```
